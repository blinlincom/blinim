# OTC交易完整流程设计

本文档描述发现页“OTC交易”的完整产品流程，参考主流 P2P/OTC 交易模式：平台托管资产，买卖双方线下付款，卖方确认收款后放币，异常订单进入申诉仲裁。

## 1. 角色

- 普通用户：可浏览广告、买入、卖出、提交付款凭证、发起申诉。
- OTC商家：完成认证并缴纳保证金后，可发布买入/卖出广告，处理订单，管理收款方式。
- 平台客服：处理申诉、冻结或解冻订单、人工放币、处罚违规用户。
- 平台管理员：配置币种、法币、支付方式、手续费、限额、商家认证规则和风控策略。

## 2. 商家认证流程

1. 用户进入“OTC交易 -> 成为商家”。
2. 填写商家资料：姓名/企业名、联系方式、常用收付款方式、经营地区、交易币种。
3. 上传认证资料：实名认证、身份证/营业执照、收款账户证明、手持证明，可按实际合规要求增减。
4. 缴纳商家保证金：后台配置保证金币种和金额，保证金冻结在平台账户。
5. 平台审核：通过后用户获得商家身份；拒绝时返回原因并允许重新提交。
6. 商家退出：无未完成订单和未结申诉时可申请退出，管理员审核后退还保证金。

## 3. 后台配置

- OTC总开关：控制发现页入口和接口访问。
- 支持币种：如 USDT、BTC、ETH、平台金币等。
- 支持法币：如 CNY。
- 支付方式：银行卡、支付宝、微信、其他自定义方式。
- 订单有效期：默认 15 分钟，可按广告单独设置。
- 放币保护期：付款后最短等待时间，降低误放风险。
- 单笔限额和每日限额：按用户等级、商家等级配置。
- 手续费：可按广告方、吃单方或平台承担配置。
- 商家保证金：金额、币种、冻结规则、解冻规则。
- 申诉超时：买方付款后卖方超时未处理，自动进入提醒或申诉。
- 风控规则：黑名单、频繁取消限制、异常付款限制、新用户限额。

## 4. 广告发布流程

商家发布广告时需要填写：

- 广告类型：我要出售 / 我要购买。
- 交易币种：例如 USDT。
- 法币：例如 CNY。
- 价格类型：固定价格 / 浮动价格。
- 固定价格：直接填写单价。
- 浮动价格：选择指数价来源并设置溢价比例。
- 总数量：广告可成交总量。
- 单笔限额：最小成交金额、最大成交金额。
- 支付方式：支持一个或多个支付方式。
- 付款时限：买家下单后必须在该时间内付款。
- 交易备注：付款说明、禁止备注敏感词、到账要求。
- 自动回复：下单后自动展示给对方的提示。
- 状态：上架、下架、暂停。

广告上架前校验：

- 商家认证通过。
- 保证金足额。
- 出售广告需要可用资产足额并冻结广告库存。
- 买入广告需要收款/付款方式完整。
- 价格、数量、限额符合后台规则。

## 5. 买入流程

以用户向商家买 USDT 为例：

1. 用户进入 OTC 首页，选择“买入”。
2. 筛选币种、金额、支付方式，选择广告。
3. 输入购买金额或数量，系统计算应付法币和可得币数量。
4. 创建订单后，平台冻结卖方对应币数量。
5. 买方在有效期内按商家收款信息线下付款。
6. 买方点击“我已付款”，上传付款凭证，可填写付款账户姓名和流水号。
7. 卖方确认到账后点击“确认收款并放币”。
8. 平台释放冻结资产给买方，订单完成。
9. 双方可评价，更新商家成交率和信用数据。

关键约束：

- 买方未点击“我已付款”且订单超时，订单自动取消，释放冻结资产。
- 买方点击“我已付款”后，卖方不能直接取消订单。
- 卖方未到账不能放币，放币后不可撤销。
- 付款必须使用本人实名账户，非本人付款可触发申诉和退款流程。

## 6. 卖出流程

以用户向商家卖 USDT 为例：

1. 用户进入 OTC 首页，选择“卖出”。
2. 选择商家买入广告。
3. 输入卖出数量或收款金额，选择自己的收款方式。
4. 创建订单后，平台冻结卖方用户的币。
5. 商家按用户收款信息线下付款。
6. 商家点击“我已付款”，上传付款凭证。
7. 用户确认到账后点击“确认收款并放币”。
8. 平台释放冻结资产给商家，订单完成。

关键约束：

- 用户作为卖方时，创建订单即冻结资产。
- 商家点击已付款后，用户不能直接取消订单。
- 用户确认收款前必须核对到账账户、金额、实名信息。

## 7. 订单状态机

- created：订单已创建，等待付款。
- paid：买方/付款方已标记付款。
- released：卖方已确认收款并放币。
- cancelled：订单已取消。
- expired：订单超时关闭。
- appeal：申诉中。
- appeal_released：申诉后平台放币。
- appeal_cancelled：申诉后平台取消。
- refunding：退款处理中。
- refunded：已退款。

状态流转：

- created -> paid：付款方点击“我已付款”。
- created -> cancelled：付款前用户主动取消。
- created -> expired：付款超时自动取消。
- paid -> released：收款方确认收款。
- paid -> appeal：任意一方发起申诉。
- appeal -> appeal_released：客服判定放币。
- appeal -> appeal_cancelled：客服判定取消。
- appeal -> refunding/refunded：客服要求退款并记录结果。

## 8. 资金托管规则

- 出售方资产在订单创建时冻结。
- 订单取消、超时或申诉取消时，冻结资产退回出售方。
- 订单完成或申诉放币时，冻结资产转给购买方。
- 平台手续费在放币时结算。
- 商家保证金单独冻结，不参与订单成交资产。
- 所有资金变化必须写入账变流水，禁止只改余额。

## 9. 申诉流程

可申诉场景：

- 买方已付款，卖方未放币。
- 卖方未收到款，买方虚假标记付款。
- 金额不一致。
- 非实名付款。
- 收款账户异常。
- 对方诱导站外交易或要求提前放币。

申诉步骤：

1. 订单进入 appeal，冻结状态保持不变。
2. 双方上传证据：付款截图、银行流水、聊天记录、退款凭证。
3. 客服介入，要求补充材料。
4. 客服判定：
   - 确认付款到账：平台放币给买方。
   - 未付款或证据不足：取消订单，资产退回卖方。
   - 付款异常需退款：订单进入退款处理中，完成后取消。
5. 记录客服处理结果、处罚、信用分变化。

## 10. 风控与处罚

- 频繁取消订单：限制下单或降低每日限额。
- 虚假付款：冻结账号、扣信用分、限制 OTC。
- 诱导站外交易：下架广告、冻结商家资格。
- 付款姓名不一致：触发人工审核。
- 大额订单：强制二次确认或延长放币保护期。
- 新用户：降低单笔限额和每日限额。
- 商家差评率过高：自动下架广告并进入复审。
- 申诉败诉过多：限制交易、扣保证金。

## 11. 合规要求

- 上线前必须确认目标运营地区是否允许相关 OTC/P2P 业务。
- 商家和大额用户必须做实名认证，必要时做企业认证。
- 需要配置 KYC、AML、反洗钱、制裁名单和黑名单策略。
- 所有订单、付款凭证、聊天、申诉、账变流水需要留痕，保留周期按当地法规配置。
- 平台不能引导用户规避银行、支付机构或当地法律规则。
- 涉及数字资产、法币兑换、支付撮合、托管或保证金时，应由法务确认业务边界。
- 文案需要明确风险提示：未确认到账禁止放币，禁止站外交易，禁止非本人实名付款。

## 12. 建议数据表

- `otc_merchant_application`：商家认证申请。
- `otc_merchant`：商家资料、状态、保证金、成交统计。
- `otc_payment_method`：用户/商家收付款方式。
- `otc_coin`：支持币种。
- `otc_fiat`：支持法币。
- `otc_advertisement`：广告。
- `otc_order`：订单主表。
- `otc_order_log`：订单状态日志。
- `otc_order_message`：订单聊天和系统通知。
- `otc_order_evidence`：付款凭证和申诉证据。
- `otc_appeal`：申诉主表。
- `otc_asset_lock`：订单资产冻结记录。
- `otc_asset_flow`：OTC账变流水。
- `otc_risk_log`：风控日志。
- `otc_review`：交易评价。

## 14. 当前已实现范围

本轮已落地第一版 OTC 核心闭环：

- 后台发现页“直播”入口已改为“OTC交易”，可通过发现页管理控制是否显示。
- 后台新增 OTC 交易管理：基础配置、支付方式配置、商家审核、广告管理、订单管理。
- 后台订单管理支持申诉订单人工处理：后台放行资产给买方，或取消订单并退回/恢复冻结资产。
- 客户端发现页 OTC 入口已接入真实交易页。
- 客户端支持申请商家、保存收款方式、买入/卖出广告列表、发布广告、创建订单、标记已付款、取消订单、确认收款并放币、发起申诉。
- 客户端 OTC 配置、商家状态、广告列表、订单列表、收款方式已写入 MMKV 缓存，进入页面先显示缓存再刷新服务器数据。
- 资金使用现有用户 `money` 字段，所有冻结、退回、到账动作都会写入账变流水。

尚未实现的增强项：

- OTC 订单聊天。
- 付款凭证图片上传和后台预览。
- 商家保证金自动冻结/退还。
- 手续费、评价、风控黑名单、每日限额。
- 独立资产账户和更细粒度冻结流水。

## 15. 当前客户端接口

所有接口沿用现有 API 加密/签名通道，公共参数仍使用 `usertoken`。

- `get_otc_config`
  - 参数：`usertoken`
  - 返回：`config`、`merchant`、`balance`

- `apply_otc_merchant`
  - 参数：`usertoken`、`merchant_name`、`contact`、`intro`
  - 作用：提交或更新商家申请，状态进入待审核。

- `get_otc_merchant_status`
  - 参数：`usertoken`
  - 作用：读取当前用户商家状态。

- `get_otc_payment_methods`
  - 参数：`usertoken`
  - 作用：读取当前用户收款方式。

- `save_otc_payment_method`
  - 参数：`usertoken`、`id`、`type`、`real_name`、`account`、`bank_name`、`qr_code`
  - 作用：新增或编辑收款方式。

- `delete_otc_payment_method`
  - 参数：`usertoken`、`id`
  - 作用：删除收款方式。

- `get_otc_home`
  - 参数：`usertoken`、`side`
  - 说明：`side=sell` 表示用户买入列表，`side=buy` 表示用户卖出列表。

- `get_my_otc_ads`
  - 参数：`usertoken`
  - 作用：读取当前用户发布的广告。

- `create_otc_ad`
  - 参数：`usertoken`、`side`、`price`、`amount`、`min_limit`、`max_limit`、`payment_types`、`terms`
  - 说明：出售广告会冻结发布方 `money` 库存。

- `update_otc_ad`
  - 参数：`usertoken`、`id`、`price`、`min_limit`、`max_limit`、`payment_types`、`terms`
  - 作用：编辑广告信息。

- `set_otc_ad_status`
  - 参数：`usertoken`、`id`、`status`
  - 说明：出售广告下架时会退回剩余冻结库存。

- `create_otc_order`
  - 参数：`usertoken`、`ad_id`、`amount`、`payment_method_id`
  - 说明：卖给商家时必须传自己的收款方式 ID，并冻结卖方 `money`。

- `get_otc_orders`
  - 参数：`usertoken`
  - 作用：读取当前用户相关订单。

- `get_otc_order_detail`
  - 参数：`usertoken`、`id`
  - 作用：读取订单详情和收款快照。

- `mark_otc_order_paid`
  - 参数：`usertoken`、`id`、`pay_evidence`
  - 作用：买方/付款方标记已付款。

- `release_otc_order`
  - 参数：`usertoken`、`id`、`payment_password`
  - 作用：卖方确认收款并放币给买方。

- `cancel_otc_order`
  - 参数：`usertoken`、`id`
  - 作用：付款前取消订单，退回或恢复冻结库存。

- `appeal_otc_order`
  - 参数：`usertoken`、`id`、`reason`
  - 作用：订单进入申诉中。

## 16. 客户端页面

- OTC 首页：买入/卖出 tab、币种筛选、金额筛选、支付方式筛选、广告列表。
- 广告详情/下单页：价格、限额、商家信息、支付方式、下单输入。
- 订单详情页：倒计时、付款信息、订单状态、聊天、凭证上传、申诉入口。
- 商家中心：认证状态、保证金、广告管理、订单管理、统计数据。
- 收付款方式管理：添加银行卡、支付宝、微信等。
- 申诉页：原因选择、证据上传、客服处理进度。

## 17. 后台页面

- OTC 概览：成交额、订单数、申诉数、商家数、风险订单。
- 基础配置：总开关、币种、法币、支付方式、手续费、限额。
- 商家审核：资料查看、通过、拒绝、冻结、退保证金。
- 广告管理：强制下架、排序、风控标记。
- 订单管理：查看订单、聊天、凭证、状态流转。
- 申诉管理：证据查看、仲裁放币、取消、退款记录、处罚。
- 风控管理：黑名单、限额、取消率规则、异常账号。

## 18. 实施顺序

1. 后台 OTC 总配置和发现页入口联动。
2. 币种、法币、支付方式基础表。
3. 商家认证和保证金冻结。
4. 广告发布和广告列表。
5. 订单创建、冻结资产、取消、超时。
6. 标记付款、确认收款并放币。
7. 订单聊天和凭证上传。
8. 申诉仲裁。
9. 风控限制、评价、统计。
10. 客户端商家中心和后台运营面板完善。
