# 搭个话客户端 UI 重设计系统说明

> 本文档只约束 Flutter 客户端 UI / 视觉框架，不改变业务功能、接口、WebRTC、IM、状态机和数据模型。

## 1. 新设计方向：Aurora Social

本轮不再是轻微美化，而是对客户端视觉系统做重新设计。

核心关键词：

- 极光感；
- 年轻社区；
- 即时通讯；
- 高对比白卡；
- 深海蓝 / 电光青 / 活力绿 / 紫色光晕；
- 功能不变，视觉表达升级。

## 2. 不可破坏的边界

允许随意重做：

- 页面背景；
- 卡片形态；
- 圆角；
- 阴影；
- 字体层级；
- 按钮视觉；
- 输入框视觉；
- 信息流排版；
- 聊天气泡；
- 导航视觉；
- 群设置、消息中心、登录页视觉框架。

禁止改动：

- API 参数；
- 登录流程；
- IM 连接与监听；
- WebRTC 通话链路；
- 信令状态机；
- 消息发送逻辑；
- 群管理业务逻辑；
- 数据模型字段；
- 页面跳转业务条件。

## 3. 视觉系统

### 3.1 色彩

主视觉来自 `BlinStyle`：

- `darkBg`：深海夜色背景；
- `green`：在线、连接、年轻感；
- `cyan`：实时消息、流动感；
- `blue`：稳定和技术信任；
- `purple`：极光识别度；
- `pink/orange`：用于少量高能状态和签到/提醒。

### 3.2 形态

- 关键入口使用极光渐变；
- 普通内容卡片使用高对比白卡；
- 页面背景使用可见但不干扰内容的 glow；
- 信息流卡片允许更强阴影和媒体视觉；
- 聊天气泡使用品牌渐变区分我方消息。

### 3.3 字体层级

- 页面大标题使用更紧凑、更强势的粗字重；
- 正文维持清晰可读；
- 标签、按钮统一高字重，增加移动端识别度。

## 4. 当前已重设计的区域

- 全局 `BlinStyle`：Aurora Social 品牌系统；
- `ThemeData`：主色切换到 cyan / purple 体系；
- 登录页：深色极光 Hero + 浮动登录卡；
- 首页顶部：社区控制台式 Hero；
- 信息流卡片：更强白卡、媒体卡片感；
- 消息中心：强化标题层级；
- 聊天页：浮动 Header、品牌渐变气泡、输入区视觉增强。

## 5. 后续继续重设计建议

优先顺序：

1. 我的页资产与功能网格；
2. 发布页编辑体验；
3. 群设置成员卡与管理弹层；
4. 聊天详情页；
5. 通话页视觉控制层。

通话页可以改按钮、布局、背景，但不能改 WebRTC renderer / peer / signal 逻辑。
